Branch : 나뭇가지
- 새로운 작업을 수행하거나 실험 변경 사항을 관리하기 위한 메커니즘
- 분리된 작업 환경을 제공
- 후에 다른 브랜치에 합쳐서 프로그램을 완성할 수 있다\
- 버전 간에 흐름 관리가 용이

※ 보통 master에 직접 작업하지 않는다
1. 새 기능(로직)을 구현해야겠다
2. branch를 새로 생성을 한다
3. 그 branch로 이동한다
4. 이동 후 그 branch에서 작업을 수행한다
5. 코드 작성하며 commit을 하며 현재 로직의 버전을 관리한다
6. 완성이 되었다
7. master에 코드를 합친다
