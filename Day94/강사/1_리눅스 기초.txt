1. 리눅스는 서버용 OS
- 서버에는 여러 계정을 생성하고
- 여러 사람이 접속해서 같이 자원을 공유하며 작업
- 따라서, 서버에 접근하기 위한 단말장치가 필요

2. 단말기
- 특정 장비를 접근 및 사용하기 위한 장치
- 서버에 접속시엔 보통 telnet, ssh등을 자주 사용

3. 리눅스는 명령 기반의 CLI(= TUI)

4. 리눅스 주요 디렉토리
- 디렉토리란? 다른 파일을 모아서 보관하는 특수 파일
- 즉, windows의 폴더를 의미

① bin	: 2진 파일로, 리눅스 기본 명령어들이 존재
② boot	: 부팅 관련 파일이 저장되어 있음
③ dev	: 장치를 관리하는 파일들이 저장
④ etc	: 그 외, 리눅스에 설치한 서비스들이 저장
⑤ home	: 일반 계정의 작업 공간들이 저장
⑥ lib	: 개발 언어 관련 라이브러리 파일들이 저장
⑦ media	: 외부 장치를 연결시킬때 사용하는 디렉토리
⑧ proc	: 실행 중인 프로그램(= 프로세스)가 저장
⑨ root	: 관리자 계정의 홈 디렉토리
⑩ sbin	: 시스템 관련 중요 명령어가 저장
⑪ tmp	: 임시 파일들이 저장되는 공간
⑫ usr	: 외부 프로그램 설치시 사용되는 공간
⑬ var	: 로그 및 메일을 저장하는 공간
⑭ /	: 시스템의 최상위를 의미. 여기 아래 모든 파일이 존재



5. 리눅스 명령어 규칙
① 명령어 [대상] 형태로 사용한다
② 명령어는 옵션을 부여해서 다양한 효과를 낼 수 있다
③ 옵션끼리는 조합이 가능하다
④ 명령어 옵션 대상은 서로 띄어쓰기로 구분한다 (콤마 x)
⑤ 대상은 파일이 될 수도 있고 디렉토리가 될 수도 있다
⑥ tab키는 자동완성이다. 꼭 쓰자
⑦ tab이 동작하지 않으면 둘 중 하나
- 명령어나 대상이 없거나
- 동일한 이름으로 시작할때 (이땐, tab을 한번 더 치면 목록이 출력됨)

⑧ 명령어는 history에 저장되고 방향키로 불러올 수 있다



6. whoami, id, who
- 접속 계정 정보를 확인 가능

7. pwd (print working directory)
- 현재 위치한 디렉토리를 출력

ex) itbank 계정으로 접속 후 입력한 결과
/home/itbank


※ /의 의미
① 제일 앞	: 최상위 위치를 의미
② 중간 	: 경로 구분자 
③ 마지막	: 디렉토리를 의미


8. su (switch user)
- 명령어 : su [계정]
- root에서 다른 계정은 패스워드 없이 바로 접속 가능
- 일반 계정에서 다른 계정은 해당 계정의 패스워드가 필요
- 계정을 생략하면 root로 접근 시도

9. exit
- 현재 접속을 끊는다

10. passwd [계정명]
- 지정 계정의 패스워드를 변경 (이건 root만 가능)
- 계정명을 생략하면 현재 계정의 pw를 변경. 
  이때, 현재 계정의 pw를 알아야함
